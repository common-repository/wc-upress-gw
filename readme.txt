== Description ==

uPress Payment Gateway adds [MAX Business](https://businesslc.max.co.il/) as a payment gateway for your WooCommerce store with a simple and fast registration process with MAX.

uPress Payment Gateway is a simple plugin which allows any user to start receiving credit card payments in a couple of button clicks.
After installation you will be forwarded to register a credit card gateway with [MAX Business](https://businesslc.max.co.il/), registration is quick and simple.
As soon as MAX approves your gateway all the required settings will be set up automatically and you will be ready to start selling.
From now on you do not have to understand technical settings, update obscure settings in multiple places, or wait for weeks for gateway approval,
uPress Payment Gateway does everything for you.

**WooCommerce is required.**

You will be prompted to register with [MAX Business](https://businesslc.max.co.il/) during setup.
[MAX Business](https://businesslc.max.co.il/) account is required in order to accept payments via this gateway.
[MAX Business Privacy Policy](https://www.max.co.il/he-IL/Pages/InformationSecurity.aspx)

Some older [YaadPay](https://yaadpay.yaad.net/) terminals are supported in collaboration with MAX.
Use of these terminals requires an active YaadPay account.
[YaadPay Privacy Policy](https://yaadpay.yaad.net/privacy-policy/)

== Installation ==

1. Upload the plugin files to the `/wp-content/plugins/wc-upress-gw` directory, or install the plugin through the WordPress plugins screen directly.
1. Activate the plugin through the 'Plugins' screen in WordPress
1. Click the link to the settings in the shown notice or go directly to WooCommerce -> Settings -> Payments -> uPress Payment Gateway
1. Click the registration link and complete the registration
1. Wait for approval notification
1. Start receiving payments!

== Frequently Asked Questions ==

= Can I use an existing MAX account? =
Yes, if you have existing terminal number and API Key just enter it in the settings, and update the MAX return URL to the URL listed in the settings (we recommend to set the error page to "view directly in payment page").
